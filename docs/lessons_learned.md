# 트러블슈팅 및 교훈 (Lessons Learned & Pitfalls)

본 문서는 L2C 프로젝트 개발 과정에서 직면했던 기술적 문제들과 이를 해결하며 얻은 교훈, 그리고 향후 유사 에이전트 시스템 설계 시 주의해야 할 함정(Pitfalls)들을 정리한 문서입니다.

---

## 1. 모니터 DPI 배율(DPI Scaling)과 Windows DWM 투명 베젤 함정

### 🚨 문제 상황
개발자 PC 및 테스트 환경에 따라 `PyAutoGUI`가 클릭을 수행할 때, 실제 브라우저의 요소가 아닌 엉뚱한 여백을 클릭하거나 오차가 누적되는 현상이 발생했습니다.

### 💡 원인 분석
1.  **DPI 배율 문제**: 윈도우 OS의 화면 배율이 100%가 아닌 125%나 150%인 경우, OS 수준의 논리 좌표와 실제 물리 해상도 간에 오프셋이 생깁니다. 스크린샷 이미지 픽셀 크기와 PyAutoGUI가 받아들이는 좌표계가 불일치했습니다.
2.  **DWM 베젤 문제**: 윈도우 DWM(Desktop Window Manager)은 창 주변에 육안으로는 보이지 않는 8픽셀 크기의 투명 베젤을 추가합니다. 이로 인해 창 경계 좌표가 실제 뷰포트 영역보다 넓게 산정되었습니다.

### 🛠️ 해결 및 교훈
*   `mss`를 통한 캡처 시 현재 모니터의 DPI 스케일 값을 동적으로 감지하여 이미지 좌표를 논리 좌표계로 역산 변환하는 오프셋 연산 로직을 탑재했습니다.
*   창 경계 계산 시 투명 베젤 8px을 강제 보정하는 코드를 작성했습니다.
*   **교훈**: OS 레벨의 GUI를 직접 조작하는 에이전트를 설계할 때에는 반드시 디바이스 픽셀 비율(Device Pixel Ratio)과 OS 창 관리자의 특수 베젤 픽셀을 고려하는 추상화 레이어가 물리 드라이버 하단에 선행되어야 함을 배웠습니다.

## 2. VLM의 시맨틱 인지 한계와 물리적 필터링의 보완

### 🚨 문제 상황
비전 에이전트가 브라우저 우측에 있는 수직 스크롤바나 웹 팝업 외곽 영역을 자꾸 클릭하려는 무한 루프에 빠졌습니다.

### 💡 원인 분석
*   VLM(Gemini 3.5 Flash)은 화면을 시각적으로 인지하지만, 웹 브라우저의 기능적 구성요소(예: 스크롤바)와 실제 웹 페이지 내부의 클릭 요소(예: 리스트 아이템)를 기능적으로 완벽히 구분하지 못합니다.
*   스크롤바 영역의 회색 바를 클릭 가능한 버튼이나 슬라이더로 판정하여 마커 ID를 부여할 경우, VLM이 스크롤을 내리기 위해 화면 상의 스크롤바 마커를 클릭하려고 시도하며 오작동을 유발했습니다.

### 🛠️ 해결 및 교훈
*   **물리적 필터링 규칙 도입**: SoM 합성 엔진(`SomEngine`)에서 화면 우측 끝 65픽셀 범위 내에 존재하는 모든 디텍션 박스는 "스크롤바" 영역으로 간주하여 마커 리스트에서 원천 강제 제외(Filtering)하는 휴리스틱 규칙을 추가했습니다.
*   **교훈**: AI 에이전트의 판단력을 100% 신뢰하기보다는, 도메인 지식에 기반한 **결정론적 룰베이스 필터링(Rule-based Filtering)**을 적절히 결합하는 것이 에이전트의 오동작을 예방하고 토큰을 절약하는 핵심 실무 기법입니다.

## 3. 루프 감지 및 자기 수정(Self-Correction) 폴백 설계의 필요성

### 🚨 문제 상황
간혹 Wanted의 로그인 유도 팝업창이나 네트워크 지연으로 인해 화면이 멈추었을 때, VLM이 동일한 액션(예: 마커 [5] 클릭)을 계속해서 반복하며 API 토큰을 낭비하는 현상이 감지되었습니다.

### 🛠️ 해결 및 교훈
*   LangGraph의 `state`에 최근 실행된 액션들의 히스토리를 유지하고, 동일한 동작이 연속으로 3회 이상 실행될 경우 강제로 루프 감지 경고를 프롬프트에 주입하거나 `go_back` 또는 페이지 새로고침을 지시하는 **자가 교정(Self-Correction) 로직**을 노드 엔진에 도입했습니다.
*   **교훈**: 자율 에이전트 시스템은 '반드시 실패한다'는 가정을 바탕으로 구축되어야 합니다. 에이전트의 추론 경로에 브레이크 역할을 하는 상위 가드레일(Guardrail) 장치 없이는 무한 루프로 인한 요금 폭탄을 피할 수 없습니다.
