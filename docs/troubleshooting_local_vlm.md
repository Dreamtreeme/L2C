# 로컬 VLM (Qwen2.5-VL) 트러블슈팅 및 성능 최적화 보고서

본 문서는 L2C 프로젝트의 비전 LLM 에이전트 구동 과정에서 로컬 VLM(Qwen2.5-VL)의 오작동 및 성능 저하 문제를 분석하고 해결한 트러블슈팅 내역을 기록합니다.

---

## 1. 개요 및 발견된 문제점

웹 브라우저 E2E 자율 제어 시나리오 진행 중, 로컬 VLM(Qwen2.5-VL 3B)에서 다음과 같은 치명적인 문제점들이 발견되었습니다.

1. **좌표 뒤틀림 및 타격 이탈 (Grounding Error)**
   - UI 요소 검출 시 실제 위치(예: 로그인 버튼, 검색 아이콘)와 수백 픽셀 이상 차이 나는 엉뚱한 여백이나 화면 중앙을 마우스가 타격하는 현상 발생.
2. **JSON 출력 붕괴 및 무한 루프 (Format Collapse)**
   - Ollama API 호출 시 `format="json"` 제약을 주었을 때, 강제 샘플링 규칙과 VLM의 추론 흐름이 충돌하여 `\n` 개행 문자나 `0`을 무한 출력하는 현상 발생.
3. **인지 대역폭 부족에 따른 요소 누락**
   - 3B 체급 모델이 1024px 이상의 고해상도 정보와 JSON 문법 제약을 함께 처리할 때, 인지 과부하(Attention Fatigue)가 걸려 1개의 요소만 추출하고 조기 종료하는 현상 발생.
4. **추론 속도 지연 (Inference Latency)**
   - 7B 체급 모델 구동 시 1회 추론에 **43~54초**가 소요되어 실시간 브라우저 제어 에이전트로서 실용성이 크게 저하됨.

## 2. 분석 결과 및 가설 검증

### 가설 1: 이미지 해상도 타협 (768px 하향 리사이징)
* **내용**: 시각 토큰 연산량을 줄이기 위해 해상도를 `1024px` $\rightarrow$ `768px`로 축소.
* **실험 결과**: **기각 (Rejected)**
  - **속도**: 46.64초 소요 (시간 단축 효과 없음).
  - **정확도**: 텍스트 및 UI 경계가 뭉개져 로그인 버튼 클릭 좌표가 왼쪽으로 356px 빗나가고 GNB 메뉴 좌표가 하단으로 처지는 등 공간 정합성이 완전히 붕괴됨.
  - **결론**: 정확한 웹 UI 제어를 위해서는 최소 **`1024px` 해상도가 물리적 마지노선**임.

### 가설 2: VRAM 절약을 통한 GPU 연산 강제 (`num_ctx` 조절)
* **내용**: `num_ctx`를 4096에서 2048로 낮춰 KV Cache VRAM 점유율을 줄임으로써 RTX 3080(10GB) 환경에서 CPU 오프로딩(Fallback)을 차단.
* **실험 결과**: **속도 지연 지속 (54.81초 소요)**
  - **이유**: 윈도우 Ollama(llama.cpp) 백엔드가 비주얼 엔코더 어텐션 연산 시 **Flash Attention 2를 사용하지 못하고 일반 연산(Quadratic Attention)**을 수행함. 이로 인해 이미지 토큰 연산(Prefill) 자체의 하드웨어 부하가 고정되어 발생함.
  - **결론**: Ollama 프레임워크 한계 하에서는 7B 체급 구동 시 약 40~50초의 속도 지연이 물리적 한계선임.

## 3. 도출된 해결책 및 반영 사항

### [해결책 A] 고정밀 하이브리드 Perception Engine 구축 (완료)
* **동작**: 평상시에는 속도가 매우 빠르고 정확한 **Gemini 3.5 Flash API**를 주력으로 활용하여 전체 시나리오를 **20~30초** 내에 마무리(API 비용 1회 완주당 약 5.4원).
* **백업**: API Key 만료나 네트워크 단절 시, VRAM 오버플로우 및 JSON 포맷 무한 루프가 해결된 로컬 **Qwen2.5-VL 7B** 모델로 자동 Fallback 구동.

### [해결책 B] 로컬 VLM 설정 고도화 (완료)
* **VRAM 최적화**: `.env`에 `OLLAMA_MODEL=qwen2.5vl:7b`를 기본 정의하되, 리사이징 기준을 **`1024px`**로 유지하여 정밀도 보장.
* **JSON 샘플러 락 해제**: `"format": "json"` 옵션을 해제하여 무한 루프 버그를 근본적으로 막고, 모델이 마크다운 코드블록 내에 유효한 JSON을 생성하도록 유도한 후 자체 **Stack Parser**로 추출 및 복제 수행.
* **추출 한계 극대화**: 검출 요소를 기존 8개에서 **25개**로 확장하여 검색 완료 후 채용 카드가 누락되는 인지 장벽 제거.

## 4. 향후 도전 과제: Direct Hugging Face 구동 전환
Ollama의 C++ 컴파일 제약(Flash Attention 미지원)을 극복하고 로컬 구동 속도를 **10초 이내**로 끌어올리기 위해, 향후 아래 단계를 추가 도전합니다.
1. **PyTorch + CUDA 기반 Hugging Face Transformers 파이프라인 연동**
2. **Windows 환경 내 `flash-attn` 라이브러리 네이티브 빌드 및 가속 활성화**
