# L2C 에이전트 트러블슈팅 가이드 (Troubleshooting Guide)

본 문서는 L2C 비전 에이전트 개발 중 발생한 핵심 문제들(좌표 부정확성, JSON 출력 오류, 로컬 VLM 인지 오차)의 발생 원인과 해결 과정을 단계별로 정리한 독립 트러블슈팅 가이드입니다.

---

## 📌 [Issue 1] 마우스 클릭 좌표 이탈 및 정합성 붕괴 (Grounding Error)

### 1. 현상
* 에이전트가 로그인 버튼(GNB 우측 상단)이나 검색창을 누르려고 할 때, 실제 대상 위치에서 수백~천 픽셀 떨어진 화면 중앙이나 빈 공간을 클릭하는 현상이 지속적으로 발생했습니다.

### 2. 원인 분석
* **경량 모델(3B)의 한계**: 로컬에서 구동한 `Qwen2.5-VL 3B` 모델은 웹 UI 요소의 미세한 경계선을 구별하는 공간 인지력이 부족하여, 요소들이 모여 있는 대략적인 구역의 중심점 또는 0, 0을 뱉어냈습니다.
* **해상도 다운샘플링 왜곡**: VRAM 제한으로 이미지를 `512px`로 줄였는데, 이 과정에서 텍스트와 아이콘이 뭉개져 VLM이 좌표를 연산할 때 원본 해상도(`1929x2044`) 복리 연산 과정에서 수십~수백 픽셀의 오차로 왜곡 증폭되었습니다.

### 3. 해결 조치
* **하이브리드 VLM 구조 도입**: 기본 Perception 분석 엔진을 **Gemini 3.5 Flash VLM**으로 전환하여 1~3픽셀 이내의 고정밀 타격을 달성하고, API 키가 없을 경우에만 로컬 VLM이 돌아가는 Fallback 아키텍처를 도입했습니다.
* **로컬 모델 세팅 상향**: 백업 구동되는 로컬 모델을 `qwen2.5vl:7b`로 증설하고, 리사이징 마지노선을 **`1024px`**로 강제하여 해상도 뭉개짐에 따른 오차를 방지했습니다.

## 📌 [Issue 2] JSON 포맷 생성 붕괴 및 무한 개행 루프 (Format Collapse)

### 1. 현상
* Ollama API 호출 시 `format="json"` 제약을 주었을 때, 응답이 정상적인 JSON이 아니라 `\n\n\n\n...` 또는 `0,0,0...` 처럼 특정 문자만 무한 반복되며 타임아웃을 유발하는 증상이 일어났습니다.

### 2. 원인 분석
* **Grammar-based Sampling의 충돌**: Ollama 추론 엔진에서 강제로 JSON 형식을 강제(샘플링 제한)하면, 소형 모델(3B/7B)은 비주얼 좌표 계산 연산과 문법 준수 연산을 동시에 수행하느라 어텐션 병목(Cognitive Load)이 생겨 문법을 잃어버리고 출력 토큰 선택 루프에 갇히게 됩니다.

### 3. 해결 조치
* **JSON 제약 해제 및 프롬프트 가이드**: Ollama 호출 단에서 `"format": "json"` 옵션을 해제하여 모델의 자유 연산을 보장하고, 마크다운 코드블록(```json ... ```) 안에 JSON을 담아 출력하도록 지시했습니다.
* **자체 Stack Parser 도입**: 정규식 파싱 대신 텍스트에서 괄호 `{` 와 `}` 의 열고 닫힘을 스택으로 계산하여 유효한 JSON 객체만 안전하게 발라내는 **Stack Parser**를 `perception.py`에 구축하여 포맷 에러를 완전히 해결했습니다.

## 📌 [Issue 3] 본문 채용공고 인식 실패 및 에이전트 무한 루프

### 1. 현상
* 검색창 입력 및 클릭은 성공했으나, 검색 결과 페이지 진입 후 에이전트가 "완료(`finish_task`)"를 선언하지 않고 계속 검색창을 다시 클릭하며 무한 루프에 도달(recursion_limit 30회 초과)했습니다.

### 2. 원인 분석
* **UI 요소 추출 한계 (8개 제한)**: 기존 Perception 엔진의 추출 제한이 최대 8개여서 상단 GNB 바(채용, 이력서, 이벤트 등)만 검출 리스트를 독차지하고, 본문에 뜬 실제 채용 공고 카드 목록은 리스트에서 완전히 밀려나 보이지 않았습니다.
* 지휘자(Gemini)는 화면에 검색 결과 공고가 나타나지 않았다고 오판하여 작업을 끝내지 못했습니다.

### 3. 해결 조치
* **추출 크기 확장**: 검출 한계를 최대 **25개**로 상향 조정했습니다. 이로써 `[신입] 데이터 분석가 채용` 같은 실제 공고 정보가 UI 상태값에 잡히게 되었고, 지휘자가 검색 성공을 확인하자마자 즉시 `finish_task`를 정상 호출하며 종료되었습니다.

## 📌 [Issue 4] 7B 로컬 모델 구동 속도 지연 (40~50초 소요)

### 1. 현상 및 분석
* `qwen2.5vl:7b`로 1024px 이미지를 로컬 스캔할 때 매번 약 45~55초가 소요되어 사용성이 저하되었습니다.
* **가설 1 (768px 하향)**은 속도 향상도 없으면서 좌표 정밀도가 극심하게 무너져 기각되었습니다.
* **원인**: 윈도우 환경 Ollama(llama.cpp) 백엔드가 비주얼 패치 연산 시 **Flash Attention 2 가속을 사용하지 않고 일반 연산(Quadratic Attention)**을 실행하기 때문에 발생하는 하드웨어 프리필(Prefill) 병목으로 판명되었습니다.

### 2. 향후 극복 방안 (Phase 3.5 신설)
* Ollama를 거치지 않고 **Python + PyTorch + CUDA 환경에서 직접 Hugging Face로 모델 가중치를 로드**하여 구동하는 방식으로 아키텍처 이전을 도전합니다.
* `flash-attn` 라이브러리를 윈도우 환경에 컴파일 설치하여 visual token prefill 연산을 **3~4배 가속(10초 이내 달성)**하고, Qwen-VL 전용 `max_pixels` 파라미터로 시각 토큰만 유연하게 다이내믹 압축하는 설정을 가동할 예정입니다.
