1. 원티드: 복사허용, 상세더보기 버튼이 존재해 크롤링 할때 버튼을 눌러야지만 전체 내용을 가져올 수 있음. 또한, 메인 페이지에서 스크롤을 하면 다음 페이지의 내용이 로드되어 이를 인지하고 스크롤 해야함. 
2. 잡코리아: 복사불가능, 상세내용이 iframe에 존재하고 여러개의 iframe이 존재해 이를 인지하고 스크롤 해야함. 메인페이지에서 스크롤을 해도 다음 페이지의 내용이 로드되지 않음.
3. 사람인: 복사불가능, 내용 자체는 텍스트이지만 jview라는 복사 방지 기능과 무거운 스크립트/광고로 인해 Playwright 등 자동화 도구 접근 시 타임아웃 및 CORS(교차 출처) 에러가 빈번하게 발생함. 상세요강이 iframe에 담겨있어 접근이 까다로우나, URL에서 고유 ID(`rec_idx`)를 추출해 순수 상세 페이지(`relay/view-detail?rec_idx=...`)로 우회 접속한 뒤, 브라우저 엔진 없이 HTTP GET 요청으로 순수 HTML을 긁어와 태그를 정규식으로 벗겨내는 방식이 가장 빠르고 안정적임.
4. 워크넷(고용24): 정부 운영 사이트로 데이터가 투명하게 공개되어 있으나, 2025년 2월부터 개인 회원은 채용정보 상세 Open-API 이용이 불가능해짐. 대신 사이트 구조가 매우 정직하고 '상세 더보기' 버튼이나 iframe을 거의 사용하지 않아 DOM 스크래핑으로도 충분히 높은 퀄리티의 텍스트 추출이 가능함. 최근 고용24(work24.go.kr)로 통합되면서 UI가 현대화되었지만 여전히 스크래핑 친화적인 구조를 유지함.
